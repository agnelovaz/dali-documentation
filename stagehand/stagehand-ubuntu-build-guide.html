Building Stagehand From Source

This guide covers building Stagehand for Ubuntu linux 14.04 (64 and 32 bit flavors).

Getting The Source Code

First clone the Stagehand source from github:

$ git clone https://github.com/gamoeba/stagehand.git

64bit Build

Getting Qt

You will need QT 5.5 (or above). This is newer than the version that aptitude will get, so you will need to download it.

Go to http://www.qt.io/download-open-source/

You can use the offline installer if you do not want to create a QT account to do this.

Download the link under Offline installers, Linux Host: Qt 5.5.1 for Linux 64-bit

$ cd ~/Downloads
$ chmod a+x ./qt-opensource-linux-x64-5.5.1.run
$ ./qt-opensource-linux-x64-5.5.1.run

When installing, be aware of the path chosen. It will typically install it within your home directory rather than in /usr/lib/x86_64-linux-gnu/ where the aptitude package lives.

Setting Up The Build

Navigate to the directory where stagehand has been cloned, then the linux build directory:

$ cd ./scripts/linux_build

You will need to tell the build scripts where you have installed Qt by setting this environment variable: This example shows the default directory used by Qt setup.

$ export QT5_64ROOT=~/Qt5.5.1/5.5/gcc_64

Building

Now run the build script:

$ ./build.sh

This will attempt to build both 64 & 32 bit versions (the 32bit build will fail and be skipped).

Create Install Packages

As the build system uses qmake, to install the binaries we create a .deb (debian) package and install it.

Run the following scripts to create packages for both the Qt (runtime) dependencies and Stagehand itself:

$ ./createqtdeps64
$ ./createstagehandupdate64

Your output directory will now contain two .deb files.

Install Packages

Install the .deb packages using dpkg like so:

$ sudo dpkg -i ./output/stagehand-qtdeps_1.02_amd64.deb
$ sudo dpkg -i ./output/stagehand_0.77_amd64.deb
$ sudo apt-get install -f

Stagehand is now installed and can be run from the command line: Note: If you require a proxy then ensure your http_proxy / https_proxy environment variables are set accordingly.

$ stagehand

32bit Build

Note: The 32bit version can be built on a 64bit machine, but will only run on a 32bit machine.

32bit Build Modifications

Follow the steps for the 64bit build with these differences:

You will need the 32bit version of Qt 5.5. Install to a different directory if you have the 64bit version installed also. You will need to set QT5_32ROOT to the 32bit Qt (instead of QT5_64ROOT). EG: export QT5_32ROOT=~/Qt5.5.1_32/5.5/gcc/

32 Build Prerequisites

You will need to install the following extra packages before running the build script:

$ sudo apt-get install gcc-4.8-multilib g++-4.8-multilib
$ sudo ln -s /usr/include/x86_64-linux-gnu/zconf.h /usr/include
$ sudo apt-get install lib32z1-dev

For older versions of ubuntu, you may need this package for zlib32 instead:

$ sudo apt-get install zlib1g:i386-dev

Building and Installing

Run the build script as before:

$ ./build.sh

It will build both the 64 and 32 bit versions if you have 64 bit Qt and env var setup also.

Run the 32bit package creation scripts:

$ ./createqtdeps32
$ ./createstagehandupdate32

The following files will be created in the output directory.

stagehand-qtdeps_1.02_i386.deb

stagehand_0.77_i386.deb

Install these in the same way as per the 64bit build.
